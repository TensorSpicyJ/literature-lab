# 拓扑序学习库宪章

## 目标

把拓扑序学习沉淀成长期可复用的知识网络，而不是零散笔记堆积。

## 范围

- topological order
- topological phase transition
- anyons / fractionalization
- quantum spin liquid
- topological superconductivity

## 阅读原则

1. 奠基文献优先
2. 经典综述与书本其次
3. 最近工作只作为补充，不替代主干

## Living Note 原则

- 概念卡是可迭代更新的 living notes，不是一次写死的定义卡。
- 后续读文献时，优先判断应不应该回写到已有概念卡，而不是默认新建一张相似卡。
- 如果新文献只是补充图像、数学语言、适用边界、反例或争议点，应优先更新原概念卡。
- 只有当新文献真正引入独立对象、稳定术语或必要中间台阶时，才拆出新的原子概念卡。
- 论文卡除了总结文章本身，还应尽量指出“这篇会更新哪些已有概念卡”。

## 形式化原则

- 成熟条目默认采用“公式优先”写法，而不是“说明文优先”写法。
- 文字只作为公式的辅助说明，不能替代公式本身。
- 知识原子应尽量写成可计算、可组合、可验证的对象。
- 正式入库、过渡草稿与旧版学习卡必须区分清楚。
- 具体约束以 [formal-kb-rules](formal-kb-rules.md) 为准。

## 输出目标

每次学习尽量沉淀成以下一种或多种对象：

- `concepts/`
- `papers/`
- `roadmap/`
- `indexes/`
- `relations/`
